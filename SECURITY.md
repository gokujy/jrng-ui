# Security Policy

## Supported releases

Security fixes are applied to the latest published JRNG UI release. Upgrade to the latest release before reporting an issue that may already be fixed.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use [GitHub private vulnerability reporting](https://github.com/gokujy/jrng-ui/security/advisories/new) and include:

- affected JRNG UI and Angular versions;
- impact and attack conditions;
- a minimal reproduction without sensitive data;
- suggested mitigation, if known.

Maintainers should acknowledge a complete report within seven days. Details remain private until a fix and coordinated disclosure are ready.

## Scope

Relevant issues include unsafe rendering, focus or interaction behavior that enables unintended actions, package-integrity problems, and security-sensitive SSR differences. General support requests belong in GitHub Discussions.
