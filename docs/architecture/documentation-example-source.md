# Documentation example source

An example definition owns its rendered preview metadata and its HTML, TypeScript, SCSS, and deterministic data. The docs renderer and code tabs read that same definition. Generated registry records are derived from canonical source and inventory, not handwritten component counts.

Examples contain local fixed data, no current timestamps, randomness, remote fonts, remote images, or automatic network calls. Heavy previews are deferred and route/code split. Responsive viewport controls are opt-in through example metadata and constrain the actual component host.

Completeness checks reject unresolved selectors, unknown inputs/outputs, placeholder code, missing backing state, duplicate slugs, missing previews/tests/API records, and preview/code divergence.
